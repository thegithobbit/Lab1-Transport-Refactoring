# Аналіз використаних інструментів розробки

Аналіз початкового коду виявив високу зачепленість (High Coupling). Було обрано підхід впровадження патерну Factory Method та інкапсуляції логіки (додавання нового типу транспорту тепер потребує зміни лише одного файлу (OCP - Open/Closed Principle).

1. **PlantUML:** Використано для генерації UML-діаграм через написаний код. Такий підхід дозволив підтримувати актуальність схем паралельно з рефакторингом коду;
2. **GitHub Actions:** Налаштовано пайплайн для автоматичного тестування, що гарантує,, що кожна зміна в коді перевіряється на цілісність. Кожен `push` у репозиторій тригерить контейнер Ubuntu, який компілює `tests.cpp` та запускає unit-тести;
3. **C++ g++ (Ubuntu Latest):** Компілятор, використаний у хмарі для перевірки працездатності коду;
4. **Markdown:** Використано для створення глосарія та звіту, що забезпечує чітку структуру документації прямо в репозиторії.
